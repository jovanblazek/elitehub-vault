# Security Policy

## Reporting a Vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Report vulnerabilities privately to:

- GitHub Security Advisories (preferred): use the repository "Report a vulnerability" flow
- Email fallback: `36740941+jovanblazek@users.noreply.github.com`

Please include:

- A clear description of the issue
- Reproduction steps or proof of concept
- Potential impact
- Suggested fix (if available)

## Disclosure Policy

- I will acknowledge reports on a best-effort basis.
- I will investigate and validate reported issues before public disclosure.
- If confirmed, I will coordinate a fix and release, then disclose details after a patch is available.

## Supported Versions

This project is in `0.x` and evolves quickly. Security fixes are applied on a best-effort basis to the latest code on `main`.
